### Siglet Visual Refinement

Iterative design review of all 50 siglet renderings, implementing several visual improvements:

- **Constrained vs unconstrained type variables**: Constrained vars (appearing in constraint args) render as colored pills; unconstrained vars render as colored text — matching the sigil rendering style
- **Record notation**: Records render as `{ ○ ○ ○ }` (one dot per field) and open rows as `( ○ ○ ○ |r )`, not a single collapsed dot
- **Applied type constructors**: Head constructor in TApp gets a larger dot (`sig-dot-head`, 11px) vs standalone types (8px) — visually distinguishing `Either String Int` as `◉ ○ ○` from `Number → Number` as `○ → ○`
- **Record(row) collapse in TApp**: `elideAST` now properly collapses `TCon "Record"` + `TRow` pairs in argument lists (fixes tailRecM3 `Step Record (row) d` rendering)
- **elideAST preserves structure**: Records and rows keep their field structure through elide (field values elided, but field count preserved for dot rendering)

## Decisions Made

- **Constrained var detection**: Collect all type variable names appearing in constraint argument positions, store in `TreeCtx.constrainedVars :: Set String`. Simple, correct, no false positives.
- **One dot per record field**: Records in siglets show field count via dots, not recursive rendering of field types. This prevents visual noise (e.g. `Map CircleId Circle` expanding to three dots instead of one).
- **Head constructor distinction**: TApp head gets `sig-dot-head` class (11px) — mirrors the conceptual distinction between type constructors (capitalized, parameterized) and concrete types (leaves of the type tree).
- **Shared examples over duplicated test data**: Single source of truth for the 50 curated signatures used by both demo and tests.
- **Surgical lock file editing over regeneration**: Spago's solver can't resolve transitive dependencies between local workspace packages when regenerating from scratch. Editing the existing lock file preserves the working resolution.
