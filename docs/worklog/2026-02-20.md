# 2026-02-20 — Counterexamples HTML Rewrite + Ecosystem Plans

## Accomplished

### Counterexamples Article: SVG → HTML + Sigil
- **Full rewrite** of `CounterexamplesGrid.purs` from a single large SVG to HTML-via-HATS with inline SVG for DAG diagrams only
- **Two-phase render**: HATS builds HTML structure with placeholder divs, then `renderClassDeclInto` injects sigil-rendered type class definitions
- **Data model enriched**: added `typeParams :: Array String` to `ClassNode`, converted method signatures from unicode arrows to `->` syntax for `parseToRenderType`
- **Simplified LayoutConfig**: removed SVG-specific fields (panelWidth, headerHeight, panelPadding, cardAreaX, cardAreaWidth); kept only DAG geometry (dagWidth, nodeW, nodeH, layerSpacing)
- HTML flow layout replaces manual Y-offset calculations (panelHeight, scanlOffsets removed)

### Dependency Upgrades
- Bumped `hylograph-selection` 0.1.1 → 0.2.0 (adds HTML element types: P, Strong, Anchor, Code, etc.)
- Bumped `hylograph-simulation` 0.1.0 → 0.2.0 (required for selection compatibility)
- Added `sigil >=0.2.0` and `language-cst-parser >=0.14.1`
- Updated `ElementType` import path in all 5 view files (`Hylograph.Internal.Selection.Types` → `Hylograph.Internal.Element.Types`)

### CSS
- Embedded full sigil.css (274 lines) into type-explorer's styles.css
- Added `--font-mono` CSS custom property
- Replaced SVG-targeting `.counterexamples-article` rules with HTML flexbox layout (panels, cards, DAG area, counterexample cards)

### Bug Fix: siblings Creates SVG `<g>` in HTML Context
- Discovered that HATS `siblings` with 2+ elements creates an SVG `<g>` wrapper, which breaks HTML rendering
- Fix: replaced `siblings (map f xs)` with direct array concatenation `(map f xs)` for all HTML contexts
- `siblings` retained only inside SVG DAG area where `<g>` is correct

## Explored But Not Pursued

- **`foreignObject` for HTML-in-SVG** — considered for embedding sigil HTML inside SVG DAG nodes. Rejected due to cross-browser sizing/hit-testing issues. This is the motivation for the sigil SVG renderer plan.

## Decisions Made

1. **Sigil needs an SVG renderer** — the HTML renderer can't be embedded in SVG DAG nodes without foreignObject. Plan written: `sigil-svg-library.md`
2. **Counterexamples belongs on a showcase site, not in CodeExplorer** — it's curated content about the PureScript ecosystem, not code exploration. Plan written: `purescript-ecosystem-site.md`
3. **TypeExplorer should dissolve back into Minard** — the useful views (force graph, matrix, grid) are just additional lenses on Minard's data. Plan written: `type-explorer-dissolution.md`
