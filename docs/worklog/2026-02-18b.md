### Inline Module Bubblepacks ‚Äî Three Iterations

1. **Kind-aggregate version**: One circle per kind (value, data, etc.), sized by `sqrt(count+1) * multiplier`. Worked but looked flat ‚Äî no visual fingerprint matching the treemap.

2. **Per-declaration version**: Imports `packDeclarations`, `kindColor`, `childKindColor` from `ModuleTreemapEnriched`. Same algorithm as the enriched treemap ‚Äî individual declaration circles with nested children (constructors inside data types, synthetic arity children for values). Semi-transparent parent circles with kind-colored strokes for containers.

3. **Race condition fix**: Declarations fetch was blocking the Halogen render cycle. Switched to `H.fork` so the fetch runs in background. `H.modify_` on completion triggers a reactive re-render ‚Äî bubblepacks fill in as data arrives. Also scoped the fetch to only annotated modules (not all modules in codebase).

### SVG Rendering Pattern

Pure Halogen SVG using `HH.elementNS` with SVG namespace ‚Äî same pattern as `ModuleSignatureMapViz`. No D3, no HATS. Each bubblepack is a self-contained `<svg>` element with computed viewBox, containing `<circle>` elements with `<title>` tooltips.

## Decisions Made

- **Per-declaration, not kind-aggregate**: The bubblepacks use the same `packDeclarations` algorithm as the enriched treemap. This gives each module a recognizable visual fingerprint that's consistent across views.
- **Left margin layout**: Bubblepacks sit in an 80px left margin gutter as margin annotations, not inline in the text flow. Each module section is a flex row with the glyph left-aligned.
- **H.fork for data loading**: "Work with Halogen, not against it" ‚Äî fork the declarations fetch so the render cycle isn't blocked. The component renders annotations immediately; bubblepacks appear reactively when declaration data arrives.
- **Scoped fetch**: Only fetch declarations for modules that actually have annotations, not the entire codebase. Much fewer HTTP requests.
- **overflow: visible on SVG**: Allows circles at the edge of the packing to render naturally without clipping.

## Explored But Not Pursued

- **D3/HATS rendering for bubblepacks**: Considered using the full HATS pipeline. Rejected ‚Äî pure Halogen SVG is simpler for static glyphs with no animation or interaction beyond tooltips.
- **Caching declaration data separately for report**: Considered a separate cache from `packageDeclarations`. Decided to reuse the shared SceneCoordinator cache ‚Äî if the user has visited a treemap first, the data is already there.
