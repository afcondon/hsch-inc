## Decisions Made

- **First backend change for anatomy**: The unused packages require a new query — can't be derived from the project's package data alone. The DB has 177 registry packages across all indexed projects; 112 are used by this project, leaving 65 unused
- **No backend changes for first 3 categories**: All three categories are computable from existing `/api/v2/packages` data (`source` field + `depends` arrays). Verified: 5 + 42 + 70 = 117 total
- **Standalone beeswarm module**: Existing `PackageSetBeeswarm.purs` (772 LOC) has coordinated highlighting, GUP scope changes, hero transitions, and git status overlays — none of which this page needs. Cleaner to have a focused ~480 LOC module reusing the same primitives
- **Flipped X-axis**: Standard beeswarm has leaves left, roots right. Anatomy flips it so workspace code reads left-to-right, parallel with the stats cards and spago.yaml blocks below
- **spago.yaml coloring matches beeswarm**: Dependency names use the same gold/blue/gray color scheme as the circles above, closing the visual loop between "what you declared" and "what you got"
- **Sunflower spiral for blob**: Golden angle packing produces a visually pleasing, non-grid cluster that naturally fills a circular area. Static rendering (no simulation) keeps it simple
- **Unused = DB complement, not full registry**: The 65 unused packages are registry packages indexed by the loader from other projects — not the full ~750 package registry. Good enough for the visual story
