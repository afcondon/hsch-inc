## Decisions Made
- **purs-backend-erl as spago backend (not two-step)**: Plan originally called for removing `backend.cmd` and running `purs-backend-erl` as a separate step. This doesn't work because without a backend, spago passes `--codegen corefn,js` to purs, which triggers JS FFI validation that fails on purerl packages. Keeping `backend.cmd` makes spago pass `--codegen corefn` only, avoiding the issue entirely.
- **`node_modules/.bin/` not `npx`**: Spago uses `spawn` (not a shell) for backend.cmd, so `npx purs-backend-erl` fails with ENOENT. Direct path to the binary works.
- **`-disable-feature maybe_expr`**: OTP 25+ made `maybe` a keyword. purs-backend-erl generates `maybe/0` and `maybe/3` from `Data.Maybe`. This erlc flag disables the maybe expression feature to allow compilation.
- **Fold nested repos, don't submodule**: The nested git repos in purescript-polyglot were leftovers from the restructuring. Small histories, code preserved — simpler to fold in than manage as submodules.
- **One big commit for restructuring**: The repo changes were coherent (all part of the same PSD3 → purescript-polyglot transformation) so a single commit was appropriate.
