# 2026-06-06 — purescript-julia: library FFI, the compute-leaf seam, and the differential test suite

## Accomplished

- **Library FFI shims** (commit `5c03755`): arrays (incl. `Data.Array.ST`,
  ST.Partial, NonEmpty.Internal), st (Internal + Uncurried), strings
  (Common/CodeUnits/CodePoints/Unsafe), foldable-traversable, integers
  (incl. Bits with JS `ToInt32` wrapping), numbers (DomainError→NaN guards
  to match JS Math), unfoldable, enums, Control.Extend. All written against
  the **real JS foreign modules** in `.spago/p/` — exported names and
  semantics matched exactly, 0-based→1-based translation at the boundary.
  Smoke test grew to 60 verified outputs.
- **Two latent bugs found while wiring**: (1) `mkFn1`/`runFn1` are
  PS-defined, not foreign — shimming them collided with the module body's
  assignment once `Data.Function.Uncurried` entered Main's closure;
  (2) `_unsafePartial(f)` must call `f(nothing)` — f is the CoreFn
  abstraction over the Partial dict; JS gets away with `f()` by arity
  tolerance, Julia cannot.
- **lorenz-leaf example** (commit `09aa8c4`): the thin-skin seam end to
  end — PS contract (`Fn4` foreign + `toJson`), Julia RK4 Lorenz numerics
  in a user `ffi-jl/` shim (first exercise of that path), dependency-free
  JSON writer using Julia multiple dispatch. 5000 points, maxZ 47.83.
- **Cross-backend differential test suite** (commit `995979a`): the
  strong-claims artifact. Ten `Test.*` modules compiled once with
  `purs --codegen corefn,js`, run on Node AND Julia, every
  `TEST name: value` line diffed. **422/426 byte-identical, 4 known
  divergences (all deliberate: 2× ASTRAL UTF-16-vs-codepoints, 2× INT64
  int32-wrap-vs-Int64), 0 failures.** `run_tests.py` is stdlib-only with
  a CI-able exit code.
- **The suite immediately caught a real bug**: `ordArrayImpl`'s length
  tiebreak was inverted for prefix-equal arrays (`compare [1,2] [1,2,0]`
  gave GT). The JS foreign returns -1 for *longer*, because the PS caller
  maps LT→1/GT→-1 and re-inverts via `compare 0 _`. Fixed.
- **JS Number formatting**: `showNumberImpl` now reproduces
  `Number.prototype.toString` placement rules (decimal within
  1e-6 ≤ |n| < 1e21, exponential with `e+`/`e-` outside, `".0"` suffix
  for integer-valued doubles) via `_js_number_string` in the runtime —
  shared with the new `Data.Number.Format` shim. All 18 formatting edge
  tests (1e10, 1e20, 1e21, 1e-6, 1e-7, max/min double, -0.0…) identical
  to JS.
- New shims: `Data.Lazy` (memoizing thunk), `Data.Number.Format`
  (Printf-based), `Data.String.Regex` (loud-failure stub so Main-less
  loaders link).

## Decisions Made

- **test-suite is foreign-free**: differential testing requires every
  module to run on both backends, so user FFI stays out; the seam demo
  lives in `examples/lorenz-leaf/` instead.
- **spago won't forward `--codegen`** — the runner resolves globs via
  `spago sources` and drives `purs compile --codegen corefn,js` directly.
- **Int overflow divergence is documented, not hidden**: `INT64-` prefixed
  tests in KNOWN_DIVERGENCES show the overflowed JS values next to the
  exact Julia ones. Depth tests restructured to count iterations (no
  overflow) so TCO-at-10⁶ is asserted as *identical* behaviour.
- Representation-level helpers belong in `PurejlRuntime` (now exports
  `_js_number_string` alongside `_runtime_lazy`), imported by every
  module header.
