## Session Focus

Implement **incremental loading** for Minard: a Sync button that re-runs the Rust loader and refreshes visualization data without a full page reload, preserving navigation context. Also fix the cache invalidation bug where lazy caches survived data reloads.

## Accomplished

### Cache Invalidation Bug Fix (SceneCoordinator)

The `Receive` handler updated `modelData`/`v2Data`/`packageSetData` but left all lazy caches stale. Now clears `packageDeclarations`, `packageCalls`, `allCallsLoaded`, `moduleAnnotations`, `allAnnotations`, `declarationStats`, `typeClassStats`, `gitStatus`, `reachabilityData`, `clusterData`, `purityData` whenever data changes. This fixes the pre-existing bug where stale data persisted after `ProjectLoaded` reloads.

### Sync Button (SceneCoordinator)

Added `RefreshPhase` ADT (`Idle | Syncing | Done | Error`) with a phase-aware button in the header bar after "Cluster":
- **Idle**: outlined "Sync" button matching existing toggle style
- **Syncing**: disabled with wait cursor
- **Done**: green checkmark, auto-clears after 1.5s
- **Error**: red, click to retry

### Refresh Data Flow (AppShell + SceneCoordinator + Loader)

Full roundtrip: Sync click -> `RequestDataRefresh` output -> AppShell forks `reloadProjects` (sequential POST `/api/v2/projects/load` per project) -> `refreshAllData` (re-fetches bulk data without changing phase) -> `RefreshDataLoaded` updates state -> SceneCoordinator `Receive` fires with new data -> caches invalidated -> viz updates in place.

Added `NotifyRefreshError` query so AppShell can signal failures back to SceneCoordinator (sets error state on button).

### Server Loader CLI Fix (Projects.js)

The server was invoking `minard-loader -q <path> <db>` but the loader requires the `load` subcommand. Fixed to: `minard-loader load -q <path> -d <db> -p <name> -l <label>`.

### DuckDB Lock Resolution (Server Main + Projects)

The Rust loader couldn't write to DuckDB because the server held an exclusive lock. Refactored `Server.Main` to hold the DB handle in an `Effect.Ref`, and `loadProject` now closes the DB before running the loader, then reopens and updates the Ref. All route handlers read from the Ref per-request.

### Anatomy Viz Refresh (ProjectAnatomyViz)

The `Receive` handler only compared `Array.length` of packages, missing changes to module counts within packages. Now also compares total module count across all packages to trigger re-render on Sync.

### Stale Test Server Discovery

Both the API server (port 3000) and frontend server (port 3001) were running from `/private/tmp/minard-test` (the clean-test clone from the earlier session), not the real repo. Diagnosed via `lsof -p <pid> | grep cwd`. Restarted both from `/Users/afc/work/afc-work/CodeExplorer/minard`.

## Decisions Made

- **Sequential `reloadProjects`**: Uses `traverse` not `parTraverse` because the server's `execSync` blocks the event loop; parallel requests would just queue
- **DB Ref pattern**: `Effect.Ref Database` in the router closure, read per-request. The loader close/reopen cycle is safe because `execSync` blocks the event loop (no concurrent requests)
- **`packageSetDataChanged` detection**: Now compares `Array.length` of packages (was only detecting `Nothing/Just` transitions, missing content changes)
- **`loadedProjects` on-demand fetch**: If `loadedProjects` is empty when Sync is clicked (user navigated away from ProjectManagement before it populated), fetch the project list first

## Explored But Not Pursued

- **Phase 2 per-project refresh**: Plan calls for a refresh button on each project card in ProjectManagementViz. Deferred — the Sync button covers the common case
- **Phase 3 freshness detection**: Polling `output/` mtimes to show a "stale" badge. More infrastructure than needed right now

## Parking Lot

- **Sync is slow (~5-10s perceived)**: The Rust loader takes ~1.6s, but the full cycle (DB close + loader + DB reopen + 4 API fetches + frontend transform) adds up. Need either a progress bar ("Syncing... 3s") or incremental loader (only process changed modules)
- **Skip loader when unnecessary**: If only annotations or view state changed, the loader POST is wasted work. Could detect whether `output/` actually changed before running
- **Search doesn't re-fire after Sync**: If the search input text is unchanged, stale results persist. Need to invalidate search results when data changes
- **Cache-busting version string**: `index.html` has `bundle.js?v=YYYYMMDD_` that must be bumped manually after each bundle. Should automate (hash-based or timestamp)
