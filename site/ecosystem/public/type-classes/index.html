The Type Class Hierarchy

The intellectual backbone of PureScript

When you need to abstract over "things that can be mapped over," reach for Functor. When you need to combine values, Semigroup. When you need sequencing with context, Monad. These aren't academic abstractions — they're the vocabulary that lets you write code once and have it work across every container, every effect type, every data structure that satisfies the contract.

PureScript inherits Haskell's type class hierarchy but makes it accessible through a clean, minimal Prelude. The classes below form a lattice: each builds on the ones above it, and understanding the hierarchy means understanding what your types can do.

The Functor Tower

The most important chain in PureScript. Every effect type, every container, every parser and decoder implements some prefix of this tower. Functor lets you transform contents. Apply lets you combine independent computations. Bind lets you sequence dependent ones. Monad unifies them.

class Functor

map transforms the value inside a context without changing the context itself. Arrays, Maybes, Effects, parser results — anything with a Functor instance lets you reach inside and transform.

class Apply

apply is the workhorse of parallel computation. When two things are independent — two API calls, two form validations, two parser branches — Apply lets you combine them without imposing an order. This is why Applicative validation can collect all errors, not just the first.

class Applicative
class Bind

bind (written >>=) is sequencing: the result of the first computation determines what the second computation is. This is strictly more powerful than Apply — you can express conditional logic, early returns, dynamic dispatch. The cost: you lose parallelism.

class Monad

Monad itself has no methods. It's the combination of Applicative and Bind — the statement that a type supports both independent combination and dependent sequencing. If something is a Monad, you can use do notation.

The Algebraic Tower

Numbers aren't special in PureScript — they're just types that happen to satisfy algebraic laws. Semiring gives you addition and multiplication. Each step up the tower adds structure: subtraction (Ring), commutativity, division (EuclideanRing), and finally Field, where everything works the way you'd expect from school arithmetic.

Eq & Ord

Equality and ordering. In PureScript, these are type class methods, not magic operators. This means you choose whether your type supports equality, and the compiler enforces it — no runtime surprises.

Semigroup & Monoid

Semigroup means "things that combine." Monoid adds an identity element. Strings, arrays, maps, functions, Maybe — the combining abstraction is everywhere once you notice it.

Foldable & Traversable

Foldable means "can be reduced to a summary value." Traversable means "can be walked with effects." Together they're the reason you can write one function that works on arrays, lists, maps, trees — anything with structure.
